# Graphify #announcements voice

Write like a human in the Discord: warm, brief, grateful. Not a press release.

## Ground truth (do not invent timelines)

- First public Graphify tag: **~April 3, 2026** (started as a Claude Code skill).
- Went viral shortly after (Karpathy `/raw` moment); grew fast after that.
- YC S26 (announced May 22, 2026).
- As of mid/late 2026 this is **months old, not years**. Never say "a year ago", "years of work", "since last year", etc. If you mention age, say something true like "a few months ago this was a tiny skill" or skip the origin story entirely.
- PyPI package is **`graphifyy`** (two y's). Repo: Graphify-Labs/graphify.

## Tone

- Natural positive. Short sentences. Sounds like someone in the server, not Marketing.
- Coolify warmth + Cursor brevity. No Mem0 hype.
- Ban: thrilled, excited to announce, supercharge, game-changer, journey, incredible milestone, momentum, "that's entirely you" essay energy, em-dash "—".
- One thank-you is enough. Don't stack gratitude paragraphs.

## CTAs (important)

- **Do not hard-sell.** No "drop a star", "if you haven't yet", "help us keep the momentum", "smash that star", growth-hack closes.
- For star milestones: celebrate + thank. Optional quiet repo link at the end (`Repo: …`) is fine; asking people to star is not.
- For tweets: link the source. For releases: point to #production-releases + release URL.

## Shape

- Greeting + ping when server-wide, then the news in **bold**, then 1–3 short lines, then thanks.
- Under ~1800 chars. One Discord message.
